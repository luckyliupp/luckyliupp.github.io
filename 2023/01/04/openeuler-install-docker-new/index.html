OpenEuler默认软件源安装docker

发表于 2023-01-04 更新于 2023-12-21 分类于 docker ， 操作系统 ， 软件安装 本文字数： 436 阅读时长 ≈ 2 分钟

一、环境及问题

操作系统环境如下：

操作系统：OpenEuler 22.03 LTS

安装方式：最小化安装

配置：firewalld、selinux保持默认开启状态，未做调整

备注：系统安装完毕后更新系统，更新命令如下：

sudo yum -y update

更新完毕后查询os版本信息如下：

# 查询命令
cat /etc/os-release

# 输出结果
NAME="openEuler"
VERSION="22.03 LTS"
ID="openEuler"
VERSION_ID="22.03"
PRETTY_NAME="openEuler 22.03 LTS"
ANSI_COLOR="0;31"

二、背景说明

本博客之前发布过一篇OpenEuler安装docker的文章，链接如下：openEuler安装docker | Hello Mr Liu (gitee.io)

该文章通过引入其他安装源的情况来进行docker的安装，docker版本为Docker version 20.10.21，经过长时间的使用，个别应用出现无法正常启动的情况，且未能排查出具体问题，谨慎起见，本篇文章介绍通过系统自带软件源进行docker的安装。

三、docker 安装

系统更新完毕后，查询软件源是否有docker：

yum search docker

查询结果如下：

docker安装：

yum install -y docker

安装过程信息：

查询docker版本和运行状态：

# 查询docker版本
docker -v       #系统自带软件源安装版本：Docker version 18.09.0, build d1134d1

# 查询docker服务状态
systemctl status docker

查询过程截图：

通过服务状态信息可以看出，docker已经启动且被设置为自启状态。

Docker-compose安装:

# docker-compose安装
yum install -y docker-compose

# docker-compose版本查询
docker-compose -v

相关截图如下：

至此，OpenEuler22.03基于系统默认软件源安装docker完成。
