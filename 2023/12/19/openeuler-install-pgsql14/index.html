安装PostgreSQL14.10并通过流复制配置主从

发表于 2023-12-19 分类于 操作系统 ， 软件安装 ， 数据库 本文字数： 1.8k 阅读时长 ≈ 7 分钟

1.环境准备

序号 IP 标识 CPU/内存配置 系统盘 数据盘
1 192.168.8.80 master 8C+16G 80G 500G
2 192.168.8.81 Slave 8C+16G 80G 500G

安装系统时，hostname分配配置为“pg-master”、”pg-slave”。

2.更新系统并安装必备组件

*（如无特殊说明代表两台机器均需要安装）

#更新系统
dnf upgrade -y

关于“dnf update” 和“dnf upgrade”的说明：

update：更新源里的软件列表。一般指量的变化，侧重更新的意思，主要是为原有的东西增加新功能，或者对已有的部分做出更改等。更新数据源。

upgrade：升级你系统里的软件。一般指质的变化，侧重升级的意思，是指从较低级版本升级到高级的版本，这种升级往往伴有新功能的增加，因此也有update的意思。

  1. 更新资源

    sudo dnf update

  2. 软件更新升级，包括升级语言包等

    sudo dnf upgrade

#安装缺失组件,OpenEuler默认最小化安装
#不同系统安装镜像默认安装的基础组件可能不一样，根据情况安装（gcc、make有的镜像安装时默认安装，有的镜像则没有默认安装）
dnf install gcc make tar readline-devel zlib-devel

3.挂载数据盘

#查看磁盘情况
fdisk -l

#给“/dev/vdb分区”
fdisk /dev/vdb

输入指令如下：

#再次查看分区情况
fdisk -l
#显示结果已创建vdb1分区
Device           Boot Start  End          Sectors     Size Id Type
/dev/vdb1        2048        1048575999   1048573952  500G 83 Linux

#格式化分区
mkfs.ext4 /dev/vdb1
#根目录下创建data挂载目录
mkdir /data
#挂载/vdb1分区到/data目录
mount /dev/vdb1 /data
#查看挂载情况（见下图挂载成功）
df -lh

#将挂载添加到启动项，否则挂载在重启后失效
vi /etc/fstab
#添加一行（见下图）
/dev/vdb1               /data                      ext4    defaults        0 0

保存关闭fstab文件，数据盘挂载完毕。

4.PostgreSQL安装

4.1下载PostgreSQL 14.10

PostgreSQL官方下载地址：PostgreSQL: File Browser

#下载PostgreSQL14.10版本
wget https://ftp.postgresql.org/pub/source/v14.10/postgresql-14.10.tar.gz

4.2编译安装

#解压源码包
tar -xzvf postgresql-14.10.tar.gz

# 切换到postgresql-14.10文件夹（系统当前在root用户根目录下）
cd postgresql-14.10
# 配置安装目录：
./configure --prefix=/data/app/pg14
# 编译（耗时较长，耐心等待）
make                             
# 安装
make install
# 创建PostgreSQL数据存放目录（可根据实际情况自行配置数据存储路径）
mkdir -p /data/appdata/pg14/data
# 创建PostgreSQL日志存放目录（可根据实际情况自行配置日志存储路径）
mkdir -p /data/applogs/pg14/logs
# 创建postgres用户
adduser postgres
# 给postgres用户授权访问data和logs目录
chown postgres /data/appdata/pg14/data
chown postgres /data/applogs/pg14/logs
# 切换到postgres用户（数据库操作不允许使用root用户）
su - postgres
# 创建日志文件【postgres用户】
cd /data/applogs/pg14/logs
touch server.log
# 初始化数据库【postgres用户】
/data/app/pg14/bin/initdb -D /data/appdata/pg14/data
# 启动数据库【postgres用户】
/data/app/pg14/bin/pg_ctl -D /data/appdata/pg14/data -l /data/applogs/pg14/logs/server.log start
# 创建test数据库【postgres用户】
/data/app/pg14/bin/createdb test  ##创建test数据库
# 连接test数据库【postgres用户】
/data/app/pg14/bin/psql test      ##连接test数据库

4.3配置PostgreSQL

以下操作如无特殊说明，均采用postgres用户（指操作系统普通用户）操作命令。

4.3.1修改数据库密码

# 我们从外部采用工具连接该数据库，需要知道数据库用户和对应的密码
# 连接默认postgres数据库；
/data/app/pg14/bin/psql      
# 修改数据库用户postgres的密码为postgres；
ALTER USER postgres WITH PASSWORD 'postgres';

4.3.2修改配置文件以支持远程连接

需要修改的文件存放于postgresql的数据存储目录（/data/appdata/pg14/data）。

# pg_hba.conf 文件
# 待修改的行：
host    all             all             127.0.0.1/32            trust
# 修改为
host    all             all             0.0.0.0/0               md5

# postgresql.conf文件
# 待修改的行
#listen_addresses = 'localhost' 
#移除前方的“#”,并将localhost修改为*
listen_addresses = '*' 

#重启postgresql
/data/app/pg14/bin/pg_ctl -D /data/appdata/pg14/data -l /data/applogs/pg14/logs/server.log restart

5.操作系统配置

5.1防火墙放行5432端口

以root用户操作。

# 配置防火墙放行5432端口
sudo firewall-cmd --zone=public --add-port=5432/tcp --permanent  
#防火墙重新加载配置文件
sudo firewall-cmd --reload

5.2环境变量配置

截止到现在如果我们想在安装postgresql的服务器使用psql或其他提供的工具连接数据库，就必须输入psql的全路径””/usr/local/postgresql/bin/psql “”来调用,并不是很方便，此时我们可以将postgresql配置到系统环境变量里面去，注意：请配置到postgres用户或者其他自定义操作postgresql的用户的bash_profile环境变量中，不要配置到root账号的环境变量中。

以postgres用户操作。

# 切换用户目录
cd ~
# 编辑用户环境变量文件
vi .bash_profile
# 将下面两行代码添加文件末尾
PATH=/data/app/pg14/bin:$PATH
export PATH
# 使配置文件生效
source .bash_profile

# 此时postgres用户就可以直接输入psql调用了

5.3开机启动配置

以root用户操作。

PostgreSQL的开机自启动配置模板在源码目录的“contrib/start-scripts”目录下,其中linux文件为linux环境下的开机自启动脚本模板，将其拷贝到“/etc/init.d”目录下，并改名为postgresql：

# 切换至源码开机脚本模版目录
cd postgresql-14.10/contrib/start-scripts/
# 拷贝linux脚本拷贝至启动目录修改名称为pgsql
cp linux /etc/init.d/pgsql
# 编辑启动脚本
vi /etc/init.d/pgsql

# 在Selinux开启Enforcing的情况，开启仍然无法启动，我们需要修复pgsql文件的安全上下文
# restorecon [选项】 文件或目录
# -R：递归.当前目录和目录下所有的子文件同时恢复；
# -V：把恢复过程显示到屏幕上；
restorecon /etc/init.d/pgsql 

# 给启动脚本增加文件执行权限
chmod a+x /etc/init.d/pgsql 

#添加至开机启动
chkconfig --add /etc/init.d/pgsql

“/etc/init.d/pgsql”需要编辑的内容如下：

# 安装位置，根据实际情况修改
prefix=/data/app/pg14

# Postgresql数据目录，根据实际情况修改
PGDATA="/data/appdata/pg14/data"

# Who to run the postmaster as, usually "postgres".  (NOT "root")
# 将启动用户修改为实际用户，这里使用postgres，所以不变。
PGUSER=postgres

# Postgresql日志文件位置，根据实际情况修改
PGLOG="/data/applogs/pg14/logs/server.log"

重启验证即可。

6.1主库配置（pg-master主机）

6.1.1创建流复制的用户

# 切换至postgres用户
[root@pg-master ~]# su - postgres

# 打开plsql
[postgres@pg-master ~]$ psql 
psql (14.10)
Type "help" for help.

# 主库创建流复制的用户
postgres=# CREATE ROLE replica login replication encrypted password 'replica';

#

6.2.2主库修改pg_hba.conf文件，允许备库IP通过复制用户访问数据库

[postgres@pg-master ~]$ vi /data/appdata/pg14/data/pg_hba.conf

# "local" is for Unix domain socket connections only
local   all             all                                     trust
# IPv4 local connections:
host    all             all             0.0.0.0/0               md5
# IPv6 local connections:
host    all             all             ::1/128                 trust
# Allow replication connections from localhost, by a user with the
# replication privilege.
local   replication     all                                     trust
host    replication     all             127.0.0.1/32            trust
host    replication     all             ::1/128                 trust
host    replication     replica         192.168.8.81/32         md5


#最后一行，添加了replica用户可以从备库IP 192.168.8.81访问主库。
