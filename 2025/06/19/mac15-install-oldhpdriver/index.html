macOS Sequoia(15.x)安装旧款打印机驱动

发表于 2025-06-19 分类于 操作系统 ， 软件安装 本文字数： 1.4k 阅读时长 ≈ 5 分钟

症状

HP打印机，型号为P1606dn，在操作系统更新到macOS Sequoia即15.x版本后，从官网下载的Mac驱动无法安装，提示如上图。也就是说HP官方目前并没有针对15.x版本制作驱动包，因为该驱动包显示的最新更新时间是2022年。

方法

下载官方驱动

通过搜索引擎搜索该打印机的mac驱动，可以找到官方的下载地址如下：

https://support.hp.com/cn-zh/drivers/hp-laserjet-pro-p1606-printer-series/model/4110409

打开后页面如下：

下载完毕后的驱动在打开时会报“本更新要求macOS v15.0或更低版本”的提示，无法安装，如下图所示。

提取驱动信息并拷贝至系统打印机驱动文件夹

根据分析，macOS的打印机驱动文件一般存在于资源库的printer文件夹下，我们尝试提取安装包的驱动文件，人工拷贝至驱动文件夹下，看看是否能够解决问题。

步骤1：

将下载的“HewlettPackardPrinterDrivers.dmg”文件双击打开后，拖动其中的“HewlettPackardPrinterDrivers.pkg”文件拖动至”下载“文件夹的”temp“文件夹（该文件夹自行创建，没有特殊含义，仅为了方便操作，无其他文件干扰）下，见下图所示：

将该文件重命名为”hp.pkg”(因接下来的步骤会产生一个HewlettPackardPrinterDrivers.pkg同名文件，避免冲突)

步骤2：

使用终端进入temp目录下：

​ 1.终端工具位于“启动台→其他”文件夹下；

​ 2.打开终端后，其默认路径在用户的根路径(终端会在命令开头显示“~”①)下，需要我们切换到下载文件夹的下的temp下②，切换完毕后通过ls命令应该可以看到我们重命名后的hp.pkg文件③，如下图所示：

然后我们接着输入命令“xar -xf hp.pkg”解压该文件，解压后的文件见下图所示：

我们需要再文件③上右键，选择”显示包内容“：

在打开的目录中选择”Payload”进行解压：

解压出来的目录需要拷贝下图中的①（“hp”）、②（“PPDs”）文件夹（可通过按住command键，选中这两个文件夹，然后按下command+c组合键进行拷贝操作，或者选中后右键选择”拷贝”）:

将上述拷贝的”hp”、“PPDs”两个文件夹粘贴到”/Library/Printers“下:

  1. 在访达图标上右键，选择”前往文件夹“，在弹出的窗口中，输入”/Library/Printers“后回车，进入到Printer文件夹；
  2. 在Printer文件夹下按下command+v组合键粘贴（或右键选择”粘贴“..）这两个文件夹即可。

测试打印机

在”系统设置“→“打印机与扫描仪”中添加时，即可正确识别驱动。

也可以点击“使用”右侧上下按钮，然后点击选择软件即可查看支持的HP打印机型号。

后记

后记1

  1. 有网友说用Pacifist for Mac来安装，经过实测，我在15.4上打开pacifist闪退报错，试了3个版本都不行，不晓得原因。
  2. 有网友说Suspicious Package这个软件也可以用来解压和拷贝安装包的文件，测试了一下，确实可以，但是因为需要安装额外的Suspicious Package软件，我这边就没有再说明了，有动手能力的可以自己试试，算是替换了步骤2。
  3. 在通过这种方式安装并成功在添加打印机时可能会弹出需要修复权限的弹窗，正常点击同意或修复就可以了（没有截到图）。
  4. 如果你的电脑上当前安装的还有其他的打印机驱动并且已经成功在使用了，那么在”步骤2“中就不能直接拷贝、粘贴两个文件夹了，品牌文件夹可以直接拷贝过去，如本文的”hp“文件夹（如果已经有同品牌的文件夹，最好把里面的文件复制过去，不要整个文件夹替换），但是”PPDs”这个文件夹，你需要拷贝“PPDs→Contents→Resources”下的所有文件，粘贴到”/Library/Printers/PPDs/Contents/Resources“下，因为如果直接拷贝、粘贴两个文件夹，它会替换原有的文件夹，导致丢失原有的其他品牌打印机的文件。

后记2

  1. 【2025-6-18】这两天重装了系统，用上述方法添加了驱动，并且添加打印机成功后，但是在打印时，弹出的打印队列中任务一直显示”已停止….，filter失败…“的字样，导致无法打印，经过十几遍的重试，逐步的排错，最后检查出来原因是需要Mac系统需要安装Rosetta，否则打印机虽然添加驱动成功，但是仍然无法使用；Rosetta这玩意一般只有在运行x86的应用程序才会提示你安装，所以我们只能通过在终端输入以下命令”softwareupdate –install-rosetta“来安装。我自己这边，当Rosetta安装成功后，就能正常打印了。
  2. 后来想了一下，其他型号或者品牌的打印机驱动如果出现这种问题，也可以参考这种方式来安装，如果有提示缺失部分软件的话，则需要考虑把驱动包里面payload里面的其他文件夹内的内容拷贝至系统对应位置后再尝试，当然一定要确保安装了Rosetta，我觉得大部分不兼容的驱动可能都还是x86的包….
