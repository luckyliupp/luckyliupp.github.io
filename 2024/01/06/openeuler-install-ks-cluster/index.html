1.2系统调优

【2024年1月11日】在kubesphere安装过程中，发现安装文件中有对系统调优的执行操作，所以这步可以不用单独做。

修改系统最大文件连接数：

[root@ks-xxx ~]# vi /etc/security/limits.conf

# 在文件末尾添加如下两行：
*               soft    nofile          65535
*               hard    nofile          65535
# 保存退出

# 重新加载系统配置
[root@ks-xxx ~]# sysctl -p
#重启生效
[root@ks-xxx ~]# reboot
# 可通过一下命令查看当前系统连接数
[root@ks-xxx ~]# ulimit -n   #同一时间最多可开启的文件数
[root@ks-xxx ~]# ulimit -a   #显示当前资源限制

#查看磁盘情况

把分区添加到物理卷(PV)

# 为`/dev/vdb1` 分区创建PV：
[root@ks-xxx ~]# pvcreate /dev/vdb1

创建新的卷组(VG)

# 如果VG还不存在，则先创建VG，否则可以将PV加到现有的VG中来扩容VG。
# 创建名为vg_ks_data 的VG，包含/dev/vdb1 PV。
[root@ks-xxx ~]# vgcreate vg_ks_data /dev/vdb1 
# 可运行vgscan来列出VG
# 运行vgdisplay vg_dlp查看VG详细信息查看VG详细信息

创建逻辑卷（LV）

# 从vg_ks_data VG中创建一个名为lv_ks_data的LV，大小为999G。
[root@ks-xxx ~]# lvcreate -L 999G -n lv_ks_data vg_ks_data
# 可运行lvs 查看LV报告或运行lvscan查看LV信息
# 运行lvdisplay /dev/vg_ks_data/lv_ks_data 可查看LV的详细信息。

将LV挂载到目录

# 创建待挂载的目录
[root@ks-xxx ~]# mkdir /app
# 挂载前先格式化LV
[root@ks-xxx ~]# mkfs.ext4 /dev/vg_ks_data/lv_ks_data
# 挂载LV
[root@ks-xxx ~]# mount /dev/vg_ks_data/lv_ks_data /app

永久挂载LV到目录

# 编辑/etc/fstab文件
[root@ks-xxx ~]# vi /etc/fstab
# 添加以下行后保存退出
/dev/vg_ks_data/lv_ks_data        /app         ext4    defaults        0 0

3.2迁移”/var/lib/docker”目录

/var/lib/docker 路径主要用于存储容器数据，在使用和操作过程中数据量会逐渐增加。因此，在生产环境中，建议根据实际需求为 /var/lib/docker 单独挂载一个大容量的磁盘。前面给我们给/app目录挂载了1T硬盘，我们在/app下创建一个存储目录，通过软链接迁移 /var/lib/docker目录。

终端下执行如下命令（三台机器都执行）：

# 停掉docker服务
[root@ks-xxx ~]# systemctl stop docker
# 创建迁移目标目录
[root@ks-xxx ~]# mkdir -p /app/docker
# 将/var/lib/docker目录改名为docker-bk，暂时备份
[root@ks-xxx ~]# mv /var/lib/docker /var/lib/docker-bk

# 创建源目录到目标目录的软链接
[root@ks-xxx ~]# ln -s /app/docker /var/lib/docker
# 查看链接是否建立，输出如下代表成功
[root@ks-xxx ~]# ll /var/lib
.....
lrwxrwxrwx.  1 root    root      11 Jan  6 15:58 docker -> /app/docker
.....
# 启动docker服务
[root@ks-xxx ~]# systemctl start docker
# 确认docker服务正常启动后删除备份目录
[root@ks-xxx ~]# systemctl status docker

4.KubeSphere集群安装

官方安装说明链接：多节点安装 (kubesphere.io)

以下操作如无特殊说明仅在ks-master01主机进行操作即可。

根据官网说明，先安装官方推荐的依赖项：socat(必须)、conntrack(必须)、ebtables(建议安装)、ipset(建议安装)，我们都安装一下：

# 依赖项(三台机器执行)
[root@xxx ~]# dnf install socat conntrack ebtables ipset -y

下载KubeKey

###### 方法1（官网）
# ①能正常访问 GitHub/Googleapis
curl -sfL https://get-kk.kubesphere.io | VERSION=v3.0.7 sh -
# ②不能正常访问 GitHub/Googleapis
export KKZONE=cn    # 重要提示：这句还是执行以下，不然后面安装也可能出现不能访问github资源的问题
curl -sfL https://get-kk.kubesphere.io | VERSION=v3.0.7 sh -

###### 方法2 😎自行找办法下载kubekey吧，下载地址
https://github.com/kubesphere/kubekey/releases

通过官网下载的会直接在目录下生成一个kk文件。自行下载的tar包需要自行解压；

# 给kk文件增加可执行权限。本文放置在了root用户的 ~/目录下。
[root@ks-master01 ~]# chmod + x kk
# 生成kubesphere集群安装配置文件
# -f 后跟配置文件路径和文件名，不加-f 和后面的路径，会默认生成一个config-sample.yaml
[root@ks-master01 ~]# ./kk create config --with-kubernetes v1.22.12 --with-kubesphere v3.4.1 -f ks-cluster-config.yaml
