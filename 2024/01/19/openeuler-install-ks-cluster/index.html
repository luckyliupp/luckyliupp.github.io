OpenEuler22.03 SP2 LTS安装Kubeshpere集群

发表于 2024-01-19 更新于 2025-06-19 分类于 操作系统 ， 软件安装 本文字数： 2.2k 阅读时长 ≈ 8 分钟

摘要：

三控制节点、三工作节点的kubesphere多节点集群安装。

1.环境规划

1.1服务器规划

序号 IP Hostname CPU/内存配置 系统盘 数据盘
1 192.168.5.31 ks-master01 8C+16G 80G 200G
2 192.168.5.32 Ks-master02 8C+16G 80G 200G
3 192.168.5.33 Ks-master03 8C+16G 80G 200G
4 192.168.5.34 ks-worknode01 16C+32G 80G 800G
5 192.168.5.35 Ks-worknode02 16C+32G 80G 800G
6 192.168.5.36 Ks-worknode03 16C+32G 80G 800G

操作系统：OpenEuler22.03 SP2 LTS

docker版本：24.0.7

kubernetes版本： v1.22.12

kubesphere版本： v3.3.1

2.挂载数据盘

**通过LV进行硬盘挂载，后续磁盘空间不足的情况下可以动态扩容。

全部机器执行

#查看磁盘情况
[root@ks-xxx ~]# fdisk -l
#给“/dev/vdb分区”并设置分区类型为Linux LVM。
[root@ks-xxx ~]# fdisk /dev/vdb

按提示输入命令：

Command (m for help): n
Partition type
   p   primary (0 primary, 0 extended, 4 free)
   e   extended (container for logical partitions)
Select (default p): p
Partition number (1-4, default 1): 回车
First sector (2048-419430399, default 2048): 回车
Last sector, +/-sectors or +/-size{K,M,G,T,P} (2048-419430399, default 419430399): 回车

Created a new partition 1 of type 'Linux' and of size 1000 GiB.

Command (m for help): t
Selected partition 1
Hex code or alias (type L to list all): 8e
Changed type of partition 'Linux' to 'Linux LVM'.

Command (m for help): w
The partition table has been altered.
Calling ioctl() to re-read partition table.
Syncing disks.

继续执行：

#再次查看分区情况
[root@ks-xxx ~]# fdisk -l
#显示结果已创建vdb1分区
Device     Boot Start        End    Sectors  Size Id Type
/dev/vdb1        2048 2097151999 2097149952 1000G 8e Linux LVM

把分区添加到物理卷(PV)

# 为`/dev/vdb1` 分区创建PV：
[root@ks-xxx ~]# pvcreate /dev/vdb1

创建新的卷组(VG)

# 如果VG还不存在，则先创建VG，否则可以将PV加到现有的VG中来扩容VG。
# 创建名为vg_ks_data 的VG，包含/dev/vdb1 PV。
[root@ks-xxx ~]# vgcreate vg_ks_data /dev/vdb1 
# 可运行vgscan来列出VG
# 运行vgdisplay vg_dlp查看VG详细信息查看VG详细信息

创建逻辑卷（LV）

# 从vg_ks_data VG中创建一个名为lv_ks_data的LV，大小为999G。
[root@ks-xxx ~]# lvcreate -L 999G -n lv_ks_data vg_ks_data
# 可运行lvs 查看LV报告或运行lvscan查看LV信息
# 运行lvdisplay /dev/vg_ks_data/lv_ks_data 可查看LV的详细信息。

将LV挂载到目录

# 创建待挂载的目录
[root@ks-xxx ~]# mkdir /app
# 挂载前先格式化LV
[root@ks-xxx ~]# mkfs.ext4 /dev/vg_ks_data/lv_ks_data
# 挂载LV
[root@ks-xxx ~]# mount /dev/vg_ks_data/lv_ks_data /app

永久挂载LV到目录

# 编辑/etc/fstab文件
[root@ks-xxx ~]# vi /etc/fstab
# 添加以下行后保存退出
/dev/vg_ks_data/lv_ks_data        /app         ext4    defaults        0 0

3.docker安装

​ KubeSpere3.3.2的安装对docker版本的要求19.3.8 +，但是OpenEuler22.03自带软件库中的docker的版本是18.x，所以我们还需要自己安装docker的最新版本（三台机器都执行）：

3.2迁移”/var/lib/docker”目录

/var/lib/docker 路径主要用于存储容器数据，在使用和操作过程中数据量会逐渐增加。因此，在生产环境中，建议根据实际需求为 /var/lib/docker 单独挂载一个大容量的磁盘。前面给我们给/app目录挂载了1T硬盘，我们在/app下创建一个存储目录，通过软链接迁移 /var/lib/docker目录。

终端下执行如下命令（全部机器都执行）：

# 停掉docker服务
[root@ks-xxx ~]# systemctl stop docker
# 创建迁移目标目录
[root@ks-xxx ~]# mkdir -p /app/docker
# 将/var/lib/docker目录改名为docker-bk，暂时备份
[root@ks-xxx ~]# mv /var/lib/docker /var/lib/docker-bk

# 创建源目录到目标目录的软链接
[root@ks-xxx ~]# ln -s /app/docker /var/lib/docker
# 查看链接是否建立，输出如下代表成功
[root@ks-xxx ~]# ll /var/lib
.....
lrwxrwxrwx.  1 root    root      11 Jan  6 15:58 docker -> /app/docker
.....
# 启动docker服务
[root@ks-xxx ~]# systemctl start docker
# 确认docker服务正常启动后删除备份目录
[root@ks-xxx ~]# systemctl status docker
● docker.service - Docker Application Container Engine

[root@ks-xxx ~]# rm -rf /var/lib/docker-bk

4.KubeSphere集群安装

官方安装说明链接：多节点安装 (kubesphere.io)

以下操作如无特殊说明仅在ks-master01主机进行操作即可。

根据官网说明，先安装官方推荐的依赖项：socat(必须)、conntrack(必须)、ebtables(建议安装)、ipset(建议安装)，我们都安装一下：

# 依赖项(三台机器执行)
[root@xxx ~]# dnf install socat conntrack ebtables ipset -y

下载KubeKey

###### 方法1（官网）
# ①能正常访问 GitHub/Googleapis
curl -sfL https://get-kk.kubesphere.io | VERSION=v3.0.7 sh -
# ②不能正常访问 GitHub/Googleapis
export KKZONE=cn    # 重要提示：这句还是执行以下，不然后面安装也可能出现不能访问github资源的问题
curl -sfL https://get-kk.kubesphere.io | VERSION=v3.0.7 sh -

###### 方法2 😎自行找办法下载kubekey吧，下载地址
https://github.com/kubesphere/kubekey/releases

通过官网下载的会直接在目录下生成一个kk文件。自行下载的tar包需要自行解压；

# 给kk文件增加可执行权限。本文放置在了root用户的 ~/目录下。
[root@ks-master01 ~]# chmod + x kk
# 生成kubesphere集群安装配置文件
# -f 后跟配置文件路径和文件名，不加-f 和后面的路径，会默认生成一个config-sample.yaml
[root@ks-master01 ~]# ./kk create config --with-kubernetes v1.22.12 --with-kubesphere v3.3.1 -f ks-cluster-config.yaml
